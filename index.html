Matrixbot API

General

Tech base

API is based on WebSocket[1][2]; technology used as a transport and JSON-RPC 2.0[3] as a communication standard. JSON-RPC-over-WebSocket allows us to use "one request - multiple answers" principle while designing our methods. This means that we're allowed to organise native subscriptions without data polling. As for end user, this means that any event that user is subscribed to, fired at the front-end immidiately, without any timeout.

Interface and method list

All the methods are grouped into interfaces so each method is named like interface.method. Interface list can be found here.

Authorization

Authorization is required for most of the methods. To perform it, use user.login method. It returns a string token.
NOTE: each token lives 7 days after its last use.
Now you can do requests with auth data. Simply add auth field to your request's parameters with the value you've got from user.login.

Arguments (parameters)

API supports only named arguments. Any request with unnamed arguments (placed in array) will be refused. So "params" field must look like {"key": "value"} and not like ["value"] or [{"key": "value"}]. Value may be omitted.

1. RFC 6455: https://tools.ietf.org/html/rfc6455
2. Protocol version: 13
3. JSON-RPC 2.0 Specification: https://www.jsonrpc.org/specification
